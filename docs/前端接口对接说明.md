# 前端接口对接说明

## 1. 启动后端服务

先训练并导出服务模型：

```bash
python scripts/train_serving_model.py
```

再启动 API：

```bash
python -m data_analysis_progress.api
```

默认地址：`http://localhost:8000`

## 2. 接口列表

### GET /health

- 用途：健康检查
- 返回示例：

```json
{
  "status": "ok"
}
```

### GET /model-info

- 用途：获取当前模型训练信息与指标
- 返回字段：
  - `metadata`: 数据集、切分方式、样本数等
  - `metrics`: validation/test 指标

### POST /predict

- 用途：单条新闻分类
- 请求体：

```json
{
  "title": "标题文本",
  "content": "正文文本",
  "meta_description": "可选",
  "summary": "可选"
}
```

- 返回：

```json
{
  "label": "reliable"
}
```

### POST /predict-batch

- 用途：批量分类（前端批量测试用）
- 请求体：

```json
{
  "texts": ["新闻文本1", "新闻文本2"]
}
```

- 返回：

```json
{
  "labels": ["fake", "reliable"]
}
```

## 3. 前端调用建议

- 当 `GET /health` 返回 `degraded` 时，提示后端模型未准备好。
- 对 `POST /predict` 的输入做最小长度校验，避免全空文本。
- 首次联调优先用 `/model-info` 确认后端模型版本和指标。
