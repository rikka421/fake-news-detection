# 假新闻检测项目技术报告（中文版）

## 1. 项目目标

本项目面向多类别假新闻识别，目标是构建一套可复现、可扩展、可本地运行的实验流程。核心任务包括：

- 构建稳定的数据子集生产流程；
- 在统一评测协议下比较传统机器学习与神经网络模型；
- 提供从快速冒烟测试到中大规模训练的分阶段实验路径；
- 输出结构化结果，便于持续对比与复验。

## 2. 数据与子集策略

数据来源为 FakeNewsCorpus 公共语料。当前版本仅保留按类别均衡采样的数据主链路：

- balanced_tiny: 10
- balanced_small: 100
- balanced_medium: 1000
- balanced_large: 10000

该策略可在最小成本下验证端到端流程，再逐步放大到更稳定的评测规模。

## 3. 模型体系

项目当前包含八类基线模型：

- 逻辑回归（TF-IDF 特征）；
- 线性支持向量机（SVM）；
- 随机森林；
- 朴素贝叶斯；
- XGBoost；
- LightGBM；
- 文本卷积网络（CNN）；
- 轻量 Transformer 编码器分类器。

模型统一接入同一套训练与评测流程，便于横向比较与后续扩展。

## 4. 实验流程设计

实验采用“先小后大”的阶段化执行：

1. 使用 balanced_tiny 做冒烟测试，确认数据读取、训练、评测链路可用；
2. 使用 balanced_small / balanced_medium 做迭代开发与参数调整；
3. 使用 balanced_large 做最终对比；
4. 将结果输出为 JSON 产物用于追踪。

## 5. 工程组织改进

本次整理后的工程结构聚焦于“可维护、可复验”：

- scripts 目录集中放置数据构建与训练入口；
- src/data_analysis_progress 作为核心可复用包；
- data/FakeNewsCorpus 保留当前有效的 balanced 子集与清单；
- docs 保留精简文档入口；
- 清理旧版冗余脚本、历史模型二进制与过时样本。

## 6. 当前结果与价值

项目已完成从数据构建到模型评测的闭环，并具备以下价值：

- 可在本地环境复现实验；
- 可快速对比不同模型族的效果与训练成本；
- 为后续引入更强模型、分组切分策略和更严格评测打下基础。

## 7. 模型性能总结

以下结果来自一次统一基准实验：

- 数据集：balanced_medium
- 样本量：1000
- 标签数：10
- 文本模式：title_content
- 防泄漏设置：mask_label_tokens
- 切分方式：grouped_title_content
- 深度模型训练轮数：3

测试集结果如下：

| 模型 | Test Accuracy | Test Macro F1 | 结论 |
|------|---------------|---------------|------|
| 随机森林 | 0.6644 | 0.6875 | 当前表现最好，类别均衡性也最好 |
| SVM | 0.6510 | 0.6593 | 次优，整体稳定，适合作为强传统基线 |
| XGBoost | 0.6443 | 0.6543 | 接近 SVM，说明 boosting 在该任务上同样有效 |
| LightGBM | 0.6174 | 0.6308 | 强于逻辑回归，弱于随机森林、SVM 与 XGBoost |
| 逻辑回归 | 0.6107 | 0.6105 | 结果稳定，仍适合快速迭代与大样本基线 |
| 朴素贝叶斯 | 0.5705 | 0.5796 | 速度快，但效果明显弱于前几类强基线 |
| CNN | 0.3490 | 0.3873 | 在当前数据规模与训练预算下不足以超过传统模型 |
| Transformer | 0.3154 | 0.3076 | 当前设置下效果最弱，说明模型容量与训练条件仍需调整 |

本轮结果说明：

- 在中等规模、分组切分且去除标签词泄漏的设定下，传统机器学习模型仍然比当前轻量深度模型更有效；
- 随机森林在本次实验中取得最高的测试准确率与宏平均 F1，可作为当前最强公开基线；
- SVM 与 XGBoost 的表现非常接近，都属于值得长期保留的强基线；
- LightGBM 的结果优于逻辑回归，但尚未超过随机森林、SVM 与 XGBoost；
- 逻辑回归虽然不是最优，但在可解释性、训练速度和稳定性方面仍然具有工程价值；
- CNN 与 Transformer 受限于当前样本规模、训练轮数和模型配置，尚未发挥优势，后续需要更长训练、更强正则化或更大数据规模验证；
- 最终上线模型选择需要同时考虑效果与实时性，不仅看单次离线指标。

## 8. CNN 与 Transformer 优化实验

为了响应“使用现有分词器/embedding 模型优化深度模型”的目标，本轮新增了可选深度输入管线：

- `tokenizer_source=simple`：原有简单分词 + 随机初始化 embedding；
- `tokenizer_source=pretrained`：使用 `google/bert_uncased_L-2_H-128_A-2` 的 BERT tokenizer 与 embedding 权重初始化，并支持冻结 embedding。

在同一实验设置下（`balanced_medium`, `epochs=3`, `grouped_title_content`, `mask_label_tokens`）对比结果：

| 模型 | Simple 管线 Macro F1 | Pretrained 管线 Macro F1 | 变化 |
|------|----------------------|--------------------------|------|
| CNN | 0.3873 | 0.4504 | +0.0631 |
| Transformer | 0.3076 | 0.1167 | -0.1909 |

结论：

- 预训练 tokenizer + embedding 初始化对 CNN 有明显增益；
- 同样设置下 Transformer 出现性能下降，说明仅替换 embedding 还不够，需要配套调参（学习率、层数、dropout、warmup、训练轮数等）；
- 后续建议将 CNN 固化为 `pretrained` 默认管线，同时对 Transformer 单独做参数搜索。

## 9. balanced_large 与 LLM 补充实验

### 9.1 balanced_large（10000）全模型结果

在 `balanced_large`、`title_content`、`mask_label_tokens`、`grouped_title_content`、`epochs=3` 条件下，测试集结果如下：

| 模型 | Test Accuracy | Test Macro F1 |
|------|---------------|---------------|
| XGBoost | 0.8072 | 0.8137 |
| LightGBM | 0.8059 | 0.8108 |
| 随机森林 | 0.7970 | 0.8015 |
| SVM | 0.7546 | 0.7566 |
| 逻辑回归 | 0.7396 | 0.7448 |
| CNN | 0.6972 | 0.7181 |
| Transformer | 0.6500 | 0.6619 |
| 朴素贝叶斯 | 0.6446 | 0.6513 |

结论：扩大到 10000 样本后，树模型（XGBoost / LightGBM / 随机森林）仍保持领先。

### 9.2 <=0.5B LLM（Qwen2.5-0.5B）实验

为评估“1 小时内处理 10k”可行性，补充了零样本 LLM 实验脚本：

- CPU（generation）：约 7.81 秒/条，10k 预计约 21.69 小时；
- 初版评测诊断发现：长文本直接 `truncation=True,max_length=512` 时，提示词尾部 `Label:` 可能被截断，导致模型更像在“续写正文”而不是“输出标签”；
- 修复后（先裁剪新闻正文 token，确保标签指令保留）：
	- GPU（RTX 3050 + cu126 + FP16，generation，200 条随机样本）平均 0.588 秒/条；
	- 吞吐约 1.70 条/秒；
	- 10k 预计约 98.08 分钟；
	- 标签提取命中率约 87.5%；
	- 准确率仍较低：Accuracy 0.070，Macro F1 0.0469。

结论：LLM 表现差并非仅来自模型能力，评测链路也存在可修复问题；但在修复后，零样本效果仍显著弱于传统模型，且速度尚未稳定达成“10k / 1小时”。
