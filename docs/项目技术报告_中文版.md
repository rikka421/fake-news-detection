# 假新闻检测项目技术报告（中文版）

## 1. 项目目标

本项目面向多类别假新闻识别，目标是构建一套可复现、可扩展、可本地运行的实验流程。核心任务包括：

- 构建稳定的数据子集生产流程；
- 在统一评测协议下比较传统机器学习与神经网络模型；
- 提供从快速冒烟测试到中大规模训练的分阶段实验路径；
- 输出结构化结果，便于持续对比与复验。

## 2. 数据与子集策略

数据来源为 FakeNewsCorpus 公共语料。为适配本地实验资源，项目采用固定规模子集策略：

- tiny: 10
- small: 100
- medium: 1000
- large: 10000

同时提供按类别均衡采样的数据版本：

- balanced_tiny: 10
- balanced_small: 100
- balanced_medium: 1000
- balanced_large: 10000

该策略可在最小成本下验证端到端流程，再逐步放大到更稳定的评测规模。

## 3. 模型体系

项目当前包含六类基线模型：

- 逻辑回归（TF-IDF 特征）；
- 线性支持向量机（SVM）；
- 随机森林；
- 朴素贝叶斯；
- 文本卷积网络（CNN）；
- 轻量 Transformer 编码器分类器。

模型统一接入同一套训练与评测流程，便于横向比较与后续扩展。

## 4. 实验流程设计

实验采用“先小后大”的阶段化执行：

1. 使用 tiny 做冒烟测试，确认数据读取、训练、评测链路可用；
2. 使用 small / medium 做迭代开发与参数调整；
3. 使用 large 或 balanced_large 做最终对比；
4. 将结果输出为 JSON 产物用于追踪。

额外提供 100k 规模的直接训练脚本，用于快速得到大样本逻辑回归基线。

## 5. 工程组织改进

本次整理后的工程结构聚焦于“可维护、可复验”：

- scripts 目录集中放置数据构建与训练入口；
- src/data_analysis_progress 作为核心可复用包；
- data/FakeNewsCorpus 保留当前有效子集与清单；
- docs 保留精简文档入口；
- 清理旧版冗余脚本、历史模型二进制与过时样本。

## 6. 当前结果与价值

项目已完成从数据构建到模型评测的闭环，并具备以下价值：

- 可在本地环境复现实验；
- 可快速对比不同模型族的效果与训练成本；
- 为后续引入更强模型、分组切分策略和更严格评测打下基础。

## 7. 模型性能总结

以下结果来自一次统一基准实验：

- 数据集：balanced_medium
- 样本量：1000
- 标签数：10
- 文本模式：title_content
- 防泄漏设置：mask_label_tokens
- 切分方式：grouped_title_content
- 深度模型训练轮数：3

测试集结果如下：

| 模型 | Test Accuracy | Test Macro F1 | 结论 |
|------|---------------|---------------|------|
| 随机森林 | 0.6644 | 0.6875 | 当前表现最好，类别均衡性也最好 |
| SVM | 0.6510 | 0.6593 | 次优，整体稳定，适合作为强传统基线 |
| 逻辑回归 | 0.6107 | 0.6105 | 结果稳定，仍适合快速迭代与大样本基线 |
| 朴素贝叶斯 | 0.5705 | 0.5796 | 速度快，但效果明显弱于 SVM 和随机森林 |
| CNN | 0.3490 | 0.3873 | 在当前数据规模与训练预算下不足以超过传统模型 |
| Transformer | 0.3154 | 0.3076 | 当前设置下效果最弱，说明模型容量与训练条件仍需调整 |

本轮结果说明：

- 在中等规模、分组切分且去除标签词泄漏的设定下，传统机器学习模型仍然比当前轻量深度模型更有效；
- 随机森林在本次实验中取得最高的测试准确率与宏平均 F1，可作为当前最强公开基线；
- SVM 的泛化表现接近随机森林，同时训练与部署复杂度较低，是非常值得保留的常用基线；
- 逻辑回归虽然不是最优，但在可解释性、训练速度和稳定性方面仍然具有工程价值；
- CNN 与 Transformer 受限于当前样本规模、训练轮数和模型配置，尚未发挥优势，后续需要更长训练、更强正则化或更大数据规模验证。

## 8. 后续工作建议

- 在 medium 与 large 上增加多随机种子评估，报告均值与方差；
- 引入更严格的数据切分策略以降低模板泄漏风险；
- 增加误分类分析与类别级指标可视化；
- 补充自动化实验报告汇总脚本。
