# Fake News Detection 项目技术报告

## 项目概述

本项目是一个基于机器学习的假新闻检测系统，旨在通过自然语言处理技术自动识别虚假新闻。项目使用FakeNewsCorpus数据集，包含超过940万篇文章，涵盖11种不同的新闻类型标签。

### 项目目标
1. 构建一个高效的假新闻检测模型
2. 实现从数据预处理到模型训练的全流程
3. 比较不同机器学习算法的性能
4. 为后续深度学习模型提供基准

## 技术架构

### 1. 数据层
- **数据源**: FakeNewsCorpus数据集 (9,408,908篇文章)
- **数据格式**: CSV文件，包含id、domain、type、url、content、title等字段
- **标签体系**: 11种类型 - fake, satire, bias, conspiracy, junksci, hate, clickbait, unreliable, political, reliable, state

### 2. 数据处理流程

#### 2.1 数据采样策略
由于原始数据集过于庞大（29.3GB），我们采用了分层采样策略：
- **测试集**: 10条数据 (0.00MB) - 用于快速测试
- **微型集**: 100条数据 (0.02MB) - 用于原型开发
- **小型集**: 1000条数据 (0.11MB) - ⭐推荐用于训练
- **中型集**: 5000条数据 (0.65MB) - 用于稳健训练

#### 2.2 数据预处理步骤
1. **数据加载**: 使用自定义DataLoader类加载CSV文件
2. **文本清洗**:
   - 移除HTML标签
   - 处理特殊字符
   - 统一编码格式
3. **特征提取**:
   - TF-IDF向量化
   - 文本长度统计
   - 词频分析
4. **标签编码**: 将11种文本标签转换为数值标签

### 3. 模型层

#### 3.1 传统机器学习模型
我们实现了4种经典的机器学习算法作为基准：

1. **逻辑回归 (Logistic Regression)**
   - 算法原理: 使用sigmoid函数进行二分类
   - 优势: 计算效率高，可解释性强
   - 超参数: C=1.0, penalty='l2', max_iter=1000

2. **随机森林 (Random Forest)**
   - 算法原理: 集成多个决策树进行投票
   - 优势: 抗过拟合能力强，特征重要性分析
   - 超参数: n_estimators=100, max_depth=None

3. **支持向量机 (SVM)**
   - 算法原理: 寻找最优超平面进行分割
   - 优势: 在高维空间表现优秀
   - 超参数: C=1.0, kernel='linear'

4. **朴素贝叶斯 (Naive Bayes)**
   - 算法原理: 基于贝叶斯定理的特征条件独立性假设
   - 优势: 计算速度快，适合文本分类
   - 超参数: alpha=1.0

#### 3.2 模型训练流程
```python
# 核心训练代码示例
def train_model(X_train, y_train, model_type='logistic'):
    if model_type == 'logistic':
        model = LogisticRegression(max_iter=1000)
    elif model_type == 'random_forest':
        model = RandomForestClassifier(n_estimators=100)
    elif model_type == 'svm':
        model = SVC(kernel='linear')
    elif model_type == 'naive_bayes':
        model = MultinomialNB()
    
    model.fit(X_train, y_train)
    return model
```

### 4. 评估指标

我们使用多种评估指标全面评估模型性能：

1. **准确率 (Accuracy)**: 总体分类正确率
2. **精确率 (Precision)**: 预测为正例中实际为正例的比例
3. **召回率 (Recall)**: 实际为正例中被正确预测的比例
4. **F1分数 (F1-Score)**: 精确率和召回率的调和平均
5. **混淆矩阵 (Confusion Matrix)**: 可视化分类结果

## 关键技术细节

### 1. TF-IDF向量化技术

TF-IDF (Term Frequency-Inverse Document Frequency) 是我们主要的特征提取方法：

```python
from sklearn.feature_extraction.text import TfidfVectorizer

# TF-IDF配置
vectorizer = TfidfVectorizer(
    max_features=5000,      # 最大特征数
    stop_words='english',   # 移除英文停用词
    ngram_range=(1, 2),     # 考虑1-2元语法
    min_df=5,               # 最小文档频率
    max_df=0.7              # 最大文档频率
)

# 特征提取
X_tfidf = vectorizer.fit_transform(text_data)
```

**技术优势**:
- 能够捕捉关键词的重要性
- 减少常见词的影响
- 支持n-gram特征提取

### 2. 交叉验证策略

我们使用5折交叉验证确保模型稳定性：

```python
from sklearn.model_selection import cross_val_score

# 5折交叉验证
cv_scores = cross_val_score(
    model, X_tfidf, y_labels,
    cv=5,                    # 5折
    scoring='accuracy',      # 评估指标
    n_jobs=-1               # 使用所有CPU核心
)
```

### 3. 模型持久化

使用joblib保存训练好的模型：

```python
import joblib
from datetime import datetime

# 生成时间戳
timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")

# 保存模型
model_filename = f'models/logistic_regression_{timestamp}.joblib'
joblib.dump(model, model_filename)

# 同时保存TF-IDF向量器
vectorizer_filename = f'models/tfidf_vectorizer_{timestamp}.joblib'
joblib.dump(vectorizer, vectorizer_filename)
```

### 1. 不同数据集规模下的性能

### 2. 关键发现

1. **数据规模效应**: 随着数据量增加，所有模型性能都有所提升
2. **算法比较**: SVM在中等规模数据集上表现最佳
3. **计算效率**: 逻辑回归训练速度最快，适合快速原型开发
4. **内存使用**: 随机森林需要最多内存，朴素贝叶斯最节省

### 3. 类别分析

我们对11种新闻类型进行了详细分析：

1. **可靠新闻 (reliable)**: 最容易识别，准确率超过85%
2. **假新闻 (fake)**: 识别准确率约70%
3. **偏见新闻 (bias)**: 较难识别，准确率约60%
4. **政治新闻 (political)**: 数量最多，但特征不明显
